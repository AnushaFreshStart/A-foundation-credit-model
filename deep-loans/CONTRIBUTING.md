# Contributing to Deeploans

Thanks for your interest in Deeploans.

This repository includes ETL pipelines, API work, and supporting apps. Small, clear contributions are the best way to get started.

## Good ways to contribute

You can help by:

- improving documentation
- reporting bugs
- suggesting small improvements
- adding simple examples
- cleaning up wording or links

## Where to look

Main repo areas:

- `etl-pipelines/`
- `api/api-backend-main/`
- `mcp-server/`

For a project overview, start with `readme.md`.

## Simple contribution flow

1. Read the relevant docs.
2. Check for an existing issue or pull request.
3. Keep your change small and focused.
4. Open an issue first if you want feedback.
5. Open a pull request with a short summary.

## Pull request tips

Before you submit, please:

- explain what changed
- explain why it helps
- update docs if needed
- check for obvious formatting or link issues

## Commit message guidance

If possible, use one of these prefixes:

- `bugfix/short-description`
- `feature/short-description`
- `docfix/short-description`
- `refactor/short-description`
- `performance/short-description`
- `test/short-description`
- `enhancement/short-description`
- `security/short-description`

Example:

```text
docfix/add-contribution-templates
```

## CLA

Please read `cla.md` before submitting a pull request. By submitting a contribution, you agree to the terms described there.

## Need help?

A good first contribution is often a docs fix, small example, or wording cleanup.
